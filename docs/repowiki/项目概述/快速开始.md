## 简介
Songloft 是一个基于 Go 和 Chi 路由器的轻量级音乐服务器，支持本地音乐文件扫描与元数据提取、歌单管理、JWT 双 Token 认证、RESTful API、Docker 容器化部署以及 WASM 插件系统。本文档提供了从零开始的完整安装与运行步骤，涵盖多种安装方式、环境准备、依赖安装、项目构建与启动过程。

## 系统要求
- **Go 版本**: 1.25.6 或更高版本
- **Flutter**: 3.29+ / Dart 3.7+（仅前端开发需要）
- **ffprobe**: 用于获取音频技术参数（可选，Docker 镜像已包含）
- **SQLite**: 内置支持，无需额外安装（使用纯 Go 实现，无需 CGO）

## 安装方式概览
Songloft 提供四种主要的安装方式，满足不同用户的需求：

### 版本选择
| 版本 | 后缀 | 说明 | 适用场景 |
|------|------|------|----------|
| 🌟 **完整版** | `-full` | 包含 Web 前端，开箱即用 | 推荐初次使用，访问服务地址即可看到 Web 界面 |
| 📦 **精简版** | 无后缀 | 不包含 Web 前端，体积更小 | 配合 Flutter 桌面/移动客户端，或前后端分离部署 |

### 平台支持
Songloft 支持以下平台和架构：
- 🐧 Linux: x86_64, ARM64, ARMv7
- 🍎 macOS: x86_64 (Intel), ARM64 (Apple Silicon)
- 🪟 Windows: x86_64, ARM64

## 方式一：直接运行二进制文件
这是最简单快捷的安装方式，适合快速体验和小型部署。

### Linux/macOS 系统
```bash
# 1️⃣ 下载对应平台的二进制文件（推荐完整版）
# 例如 Linux x86_64 完整版:
wget https://github.com/songloft-org/songloft/releases/latest/download/songloft-linux-amd64-full

# 2️⃣ 添加执行权限
chmod +x songloft-linux-amd64-full

# 3️⃣ 创建必要目录
mkdir -p music data

# 4️⃣ 运行
./songloft-linux-amd64-full
```

### Windows 系统
```powershell
# 1️⃣ 下载对应平台的二进制文件（推荐完整版）
# 例如 Windows x86_64 完整版: songloft-windows-amd64-full.exe

# 2️⃣ 创建必要目录
mkdir music
mkdir data

# 3️⃣ 运行
.\songloft-windows-amd64-full.exe
```

### 命令行参数
```bash
# 查看帮助
./songloft-linux-amd64 -help

# 指定端口
./songloft-linux-amd64 -port 8080

# 指定管理员用户名
./songloft-linux-amd64 -username admin

# 指定管理员密码
./songloft-linux-amd64 -password your_password

# 指定数据库文件路径
./songloft-linux-amd64 -db data/songloft.db
```

## 方式二：Docker 部署
Docker 部署提供了标准化的容器化解决方案，适合生产环境和需要隔离的场景。

### 从 Docker Hub 拉取（推荐）
```bash
# 🌟 拉取完整版镜像（推荐，包含 Web 前端）
docker pull songloft/songloft:full

# 或指定版本的完整版
docker pull songloft/songloft:1.0.0-full  # 替换为具体版本号

# 📦 拉取精简版镜像（不含 Web 前端）
docker pull songloft/songloft:latest
# 或 docker pull songloft/songloft:1.0.0

# 运行容器（以完整版为例）
docker run -d \
  --name songloft \
  -p 58091:58091 \
  -v /path/to/music:/app/music \
  -v /path/to/data:/app/data \
  -e ADMIN_USERNAME=admin \
  -e ADMIN_PASSWORD=your_secure_password \
  songloft/songloft:full
```

### 从 Release 导入镜像
```bash
# 1️⃣ 下载对应平台的 Docker 镜像 tar 文件（推荐完整版）
wget https://github.com/songloft-org/songloft/releases/latest/download/songloft-docker-linux-amd64-full.tar

# 2️⃣ 导入镜像
docker load -i songloft-docker-linux-amd64-full.tar

# 3️⃣ 查看导入的镜像
docker images | grep songloft

# 4️⃣ 运行容器（使用导入的镜像标签）
docker run -d \
  --name songloft \
  -p 58091:58091 \
  -v /path/to/music:/app/music \
  -v /path/to/data:/app/data \
  -e ADMIN_USERNAME=admin \
  -e ADMIN_PASSWORD=your_secure_password \
  songloft/songloft:full
```

## 方式三：Docker Compose 部署
Docker Compose 提供了更简洁的服务编排方式，特别适合需要多个服务协同工作的场景。

### 基础配置
创建 `docker-compose.yml` 文件：
```yaml
services:
  songloft:
    image: songloft/songloft:full
    container_name: songloft
    restart: always
    ports:
      - "58091:58091"
    volumes:
      - ./music:/app/music
      - ./data:/app/data
    environment:
      - ADMIN_USERNAME=admin
      - ADMIN_PASSWORD=your_secure_password
```

启动服务：
```bash
# 启动（后台运行）
docker compose up -d

# 查看日志
docker compose logs -f songloft

# 停止服务
docker compose down
```

### 高级配置（精简版镜像）
如果需要使用精简版镜像并自定义更多参数：
```yaml
services:
  songloft:
    image: songloft/songloft:latest
    container_name: songloft
    restart: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - /path/to/your/music:/app/music
      - songloft-data:/app/data
    environment:
      - ADMIN_USERNAME=admin
      - ADMIN_PASSWORD=your_secure_password
      - LISTEN_PORT=8080

volumes:
  songloft-data:
```

## 认证使用指南
Songloft 使用 JWT 双 Token 认证机制，提供安全的 API 访问控制。

### 登录获取 Token
```bash
curl -X POST http://localhost:58091/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"your_secure_password"}'
```

响应示例：
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expires_in": 604800,
  "token_type": "Bearer"
}
```

### 使用 Access Token 访问受保护的接口
```bash
curl -X GET http://localhost:58091/api/v1/songs \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

curl -X GET http://localhost:58091/api/v1/playlists \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

### 刷新 Token
当 Access Token 过期时，使用 Refresh Token 获取新的 Token：
```bash
curl -X POST http://localhost:58091/api/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refresh_token":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}'
```

### 登出
撤销当前会话的所有 Token：
```bash
curl -X POST http://localhost:58091/api/v1/auth/logout \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## 常见问题解答
### 端口冲突问题
如果 58091 端口被占用，可以通过环境变量或命令行参数修改：
```bash
# 使用环境变量
export LISTEN_PORT=8080
./songloft-linux-amd64-full

# 或使用命令行参数
./songloft-linux-amd64-full -port 8080
```

### 权限问题
确保音乐目录和数据目录具有正确的读写权限：
```bash
# 创建目录并设置权限
mkdir -p music data
chmod -R 755 music
chmod -R 755 data
```

### Docker 权限问题
使用 Docker 时确保卷挂载路径正确：
```bash
# 使用绝对路径挂载
docker run -v /absolute/path/to/music:/app/music -v /absolute/path/to/data:/app/data ...

# 或使用相对路径
docker run -v $(pwd)/music:/app/music -v $(pwd)/data:/app/data ...
```

### 音频文件识别问题
如果某些音频文件无法识别，检查文件格式支持：
- 支持格式：MP3、FLAC、WAV、APE、OGG、M4A 等
- 确保文件没有损坏
- 检查文件编码和元数据

## 故障排除
### 服务启动失败
1. **检查端口占用**：确认 58091 端口未被其他程序占用
2. **检查权限**：确保对 music 和 data 目录有读写权限
3. **查看日志**：使用 `docker compose logs -f songloft` 查看详细错误信息

### 音乐扫描失败
1. **检查文件格式**：确保音乐文件格式受支持
2. **检查文件权限**：确保程序可以读取音乐文件
3. **检查磁盘空间**：确保有足够的磁盘空间进行文件扫描

### API 认证失败
1. **检查 Token 有效期**：Access Token 通常有效期为 7 天
2. **使用 Refresh Token**：当 Access Token 过期时使用 Refresh Token
3. **检查请求头格式**：确保 Authorization 头格式正确

### Docker 镜像拉取失败
1. **检查网络连接**：确保可以访问 Docker Hub
2. **使用国内镜像**：配置 Docker 镜像加速器
3. **手动导入镜像**：从 Release 页面下载镜像文件并导入
