## 简介
Songloft 是一个前后端分离的音乐管理系统，采用 Go 后端 + Flutter 前端 + WASM 插件系统的架构设计。项目提供了本地音乐管理、网络歌曲、电台播放和歌单管理等核心功能，同时通过插件系统实现高度可扩展的功能模块。

### 安全认证体系
系统采用 JWT 双 Token 认证机制，提供完整的安全防护：

```mermaid
flowchart TD
Start([用户登录]) --> Validate[验证凭据]
Validate --> Valid{凭据有效?}
Valid --> |否| Error[返回错误]
Valid --> |是| GenAccess[生成Access Token<br/>7天有效期]
GenAccess --> GenRefresh[生成Refresh Token<br/>30天有效期]
GenRefresh --> SaveDB[保存到数据库]
SaveDB --> Cache[内存缓存]
Cache --> Success[返回认证信息]
subgraph "Token验证流程"
Verify[接收Token] --> CheckCache[检查缓存]
CheckCache --> CacheHit{缓存命中?}
CacheHit --> |是| Return[返回Claims]
CacheHit --> |否| ParseToken[解析JWT签名]
ParseToken --> CheckDB[检查数据库撤销状态]
CheckDB --> Revoked{已撤销?}
Revoked --> |是| Deny[拒绝访问]
Revoked --> |否| CacheUpdate[更新缓存]
CacheUpdate --> Return
end
```

## 性能考虑
- **压缩传输**: 启用 Gzip 压缩中间件，减少静态资源传输体积
- **缓存策略**: 静态资源长期缓存，API 请求短时缓存
- **连接池**: 数据库连接池管理，避免频繁连接开销
- **插件隔离**: WASM 实现插件隔离，防止插件影响主程序稳定性
- **内存管理**: JS 运行时环境按需创建和销毁，避免内存泄漏

## 故障排除指南

### 常见问题诊断
1. **认证失败**: 检查 JWT 密钥生成和缓存机制
2. **插件加载失败**: 验证 WASM 文件完整性，检查插件元数据
3. **数据库连接问题**: 确认数据库文件路径和权限
4. **跨域问题**: 配置 CORS 策略，验证来源域名

### 日志和监控
- 使用 slog 结构化日志记录
- Tracely 监控错误和性能指标
- Panic 捕获和上报机制

## 结论
